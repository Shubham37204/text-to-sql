## Features

- **Natural language to SQL** — Type a question, get a SQL query generated by Llama 3 via Groq
- **Conversation memory** — Ask follow-up questions; the assistant remembers your session context
- **Query history** — Every query is saved to PostgreSQL and displayed in the sidebar
- **Auto chart generation** — Numeric results automatically render as bar or line charts
- **Export to CSV** — Download any result set with one click
- **Safety guardrails** — Only `SELECT` queries are allowed; no destructive operations possible
- **Schema-aware** — The LLM sees your real table structure before generating any SQL

## How It Works

### Request Flow

```
┌─────────────────────────────────────────────────────────────┐
│ 1. User asks question in frontend (HTMX form POST)          │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│ 2. FastAPI receives request @POST /api/query                │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│ 3. SQLAlchemy reflects live database schema                 │
│    (table names, columns, data types)                       │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│ 4. Build system prompt with schema + conversation history   │
│    Send to Groq API (Llama 3 70B)                           │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│ 5. LLM returns generated SQL query                           │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│ 6. Safety checks:                                            │
│    ✓ Only SELECT queries allowed?                           │
│    ✓ All referenced tables exist?                           │
│    ✓ No INVALID_QUERY tokens?                               │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│ 7. Execute SQL on PostgreSQL                                │
│    Save query + results to query_history table              │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│ 8. Analyze results:                                         │
│    • Numeric data? → Auto-generate Chart.js config         │
│    • Large result set? → Paginate or limit rows            │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│ 9. Render HTML fragment + inject via HTMX                  │
│    (no page reload needed)                                  │
└─────────────────────────────────────────────────────────────┘
```

### Key Technical Decisions

- **Dynamic schema reflection** — No manual schema definition; detects tables on every request
- **Conversation memory** — Server-side session stores past queries for context (LLM can refer to them)
- **In-memory caching** — Session data cached to reduce DB calls (resets on server restart)
- **HTMX injection** — HTML responses injected directly into DOM (fast, interactive)


## Getting Started

### Prerequisites

Before you start, ensure you have:

- **Python 3.11+** — [Download here](https://www.python.org/downloads/)
- **Git** — [Download here](https://git-scm.com/)
- **Groq API Account** — Free at [console.groq.com](https://console.groq.com) (requires email)
- **Neon PostgreSQL Account** — Free at [neon.tech](https://neon.tech) (GitHub/Google sign-in)


### 1. Clone the repository

```bash
git clone https://github.com/yourusername/text-to-sql.git
cd text-to-sql
```

### 2. Install dependencies

```bash
# Create virtual environment (recommended)
python -m venv venv

# Activate it
# On Windows:
venv\Scripts\activate
# On macOS/Linux:
source venv/bin/activate

# Install packages
pip install -r requirements.txt
```

### 3. Set up environment variables

Copy the example file and edit it:

```bash
cp .env.example .env
```

Open `.env` in your editor and fill in:

```env
GROQ_API_KEY=your_groq_api_key_here
DATABASE_URL=postgresql://user:password@ep-xxxx.us-east-2.aws.neon.tech/dbname?sslmode=require
GROQ_MODEL=llama3-70b-8192
```

### 4. Run the server

```bash
uvicorn app.main:app --reload
```


## Configuration

### Environment Variables

| Variable | Required | Description | Example |
|---|---|---|---|
| `GROQ_API_KEY` | ✅ Yes | API key from [Groq Console](https://console.groq.com) | `gsk_...` |
| `DATABASE_URL` | ✅ Yes | PostgreSQL connection string (Neon) | `postgresql://user:pass@...` |
| `GROQ_MODEL` | No | LLM model to use (default: `llama3-70b-8192`) | `llama3-70b-8192` |

### Getting Your Keys

1. **Groq API Key:**
   - Sign up at [console.groq.com](https://console.groq.com)
   - Navigate to **API Keys**
   - Create and copy your key

2. **PostgreSQL Connection (Neon):**
   - Create a free database at [neon.tech](https://neon.tech)
   - Copy the connection string from the **Connection Details** panel
   - Append `?sslmode=require` if not present

### Sample Queries

| Question | SQL Pattern | Use Case |
|---|---|---|
| How many users signed up this month? | `COUNT` + `WHERE` + date filter | Time-based aggregation |
| Show me the top 5 products by total revenue | `ORDER BY DESC` + `LIMIT` | Top-N queries |
| List all orders with value greater than 1000 | `WHERE` clause | Filtering |
| Show each user's name with their total orders | `JOIN` + `GROUP BY` | Joins and aggregation |
| What's the average order value by month? | `DATE_TRUNC` + `GROUP BY` + `AVG` | Time-series analysis |
